# Code map — `index.html`

Everything is in one file, in three parts: `<style>`, the `<body>` markup, and one
`<script>` IIFE. Anchors below are stable names (CSS comment headers, function names,
variable names) rather than line numbers, which drift. Grep for the quoted anchor.

## `<style>` — CSS sections

Each block starts with a `/* ---------- Name ---------- */` comment:

| Anchor | Covers |
|---|---|
| `Header` | Title, language toggle, Major/Minor power-factor toggle |
| `Timer readout (sticky)` | The big sticky hit-factor readout + the 4 stat tiles below it |
| `Sections` | `.section-label` headings |
| `Target cards (2-column grid)` | `.grid`, `.card`, `.card-top`, `.zlab`, `.zval`, `.stepper`, `.count`. **The odd-last-card span rule lives here:** `.grid > .card:last-child:nth-child(odd)` |
| `Time` | Numeric time input + range slider styling |
| `Calculation breakdown` | `.calc-card` and its lines |
| `History` | `.save` button, `.history` list (`.hist-item`, `.hist-hf`, `.hist-sub`, `.hist-actions`), `.clear-history` |
| `Reset` | Reset button + `.footnote` |

## `<body>` — markup

Static shell only; the input cards are generated by JS (see `makeRow`). Landmarks by id /
`data-i18n`:

- `.readout` → `#hf` (hit factor), and stat tiles `#score`, `#maxPts`, `#pct`, `#rounds`.
- `#zoneRows.grid` and `#penaltyRows.grid` — empty containers filled at init.
- Time card → `#timeNum` (number) + `#timeSlider` (range).
- `#calc` — the calculation-breakdown card, filled by `render()`.
- `#reset`, then the History section: `#save` (save/update button) + `#history` (list
  container filled by `renderHistory()`; its rows and the `#clearHistory` button are
  generated by JS), and `.footnote` (text injected from i18n).

## `<script>` — the IIFE

Order top to bottom:

1. **`I18N`** — the translation dictionary, `{ sv: {...}, en: {...} }`. Every user-facing
   string lives here. Sub-objects: `ariaNames` (per-key accessible labels for steppers).
2. **`state`** — `{ lang, pf, counts, time, history, editingId }`. `counts` keys:
   `A, C, D, S` (hits) and `M, NS, P` (penalties: miss, no-shoot, procedure). **No `F`** —
   FTE was removed. `history` is the saved-stage list; `editingId` is the id of the entry
   currently loaded for editing, or `null`.
3. **`ZONE_PTS`** — `{ major:{A,C,D,S}, minor:{A,C,D,S} }`. `PENALTY = 10`.
4. **`T()`** — returns the active language's dictionary (`I18N[state.lang]`).
5. **`storedLang` / `storeLang`** — localStorage read/write, try/catch-guarded.
   **`storedHistory` / `storeHistory` / `sanitizeHistory` / `copyCounts`** — same
   try/catch contract for the `ipsc-history` key; `sanitizeHistory` drops malformed
   entries on load.
6. **`zoneDefs` / `penaltyDefs`** — the data that drives the grids. Each entry:
   `{ key, label, small?, zoneI18n? }`. `key` must match a `state.counts` key.
7. **`makeRow(def, isPenalty)`** — builds one `.card` (label + value + stepper) from a def.
   Element ids follow the pattern `cnt<KEY>` (the count) and `val<KEY>` (the point value).
8. **`calc()`** — pure function. Returns `{ score, raw, rounds, hf }`. `score = max(0, raw)`.
9. **`buildCalc(r)`** — returns the HTML string for the breakdown card (points expression
   and the `HF = score ÷ time` line, plus a floor note when `raw < 0`).
10. **`render()`** — the single source of DOM updates. Recomputes `calc()`, updates every
    count/value/readout/stat and the `#calc` card. **Call `render()` after any state
    change.**
11. **`applyLang(lang)`** — sets language, persists it, rewrites all `[data-i18n]` and
    `[data-aria]` nodes, updates toggle active states, then `render()`.
12. **History** — `saveStage` (create or, when `editingId` is set, update an entry),
    `editStage` (restore an entry's inputs and enter edit mode), `removeStage`,
    `clearHistory` (confirm-guarded), `renderHistory` (rebuilds `#history`; interpolates
    only numbers and controlled i18n labels), and `updateSaveButton` (Save↔Update label).
13. **Event wiring** — steppers (`applyDelta` + pointer press-and-hold via `clearHold`,
    plus keyboard `click` with `e.detail === 0`), time input/slider sync (`syncTimeInputs`),
    power-factor toggle (`setPF`), save button, `#history` delegated clicks, reset, and
    init (history + language from storage or `navigator.language`).

## Checklist: add a new target/penalty row

1. Add `{ key:"X", label:"..." }` to `zoneDefs` or `penaltyDefs`.
2. Add `X:0` to `state.counts` (init) **and** the reset object in the reset handler.
3. If it scores points, add `X` to both `ZONE_PTS.major` and `ZONE_PTS.minor`, and include
   it in `calc()` (`raw` and, if it's a required hit, `rounds`).
4. Add it to the penalty sum in `calc()` and `buildCalc()` if it's a penalty.
5. Add an `ariaNames.X` entry in **both** `sv` and `en`.
6. If it needs a translated label (like steel), add an i18n key and set `zoneI18n` on the
   def; otherwise the literal `label` is shown.
7. Update the `footnote` strings (both languages) and `README.md` scoring table.
8. Check the grid still fills cleanly (odd counts auto-span the last card).
